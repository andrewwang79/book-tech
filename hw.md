# 硬件开发
## 开发角色
| 角色 | 职责 | 开发语言 | 说明 |
| - | - | - | - |
| 硬件工程师 | 硬件电路设计 | 电路设计工具 | 原理图，PCB |
| FPGA工程师 | 硬件逻辑设计与实现 | Verilog/VHDL | 硬件逻辑、电路的“软件化”(用代码画电路) |
| BSP工程师 | 操作系统与硬件的桥梁搭建 | C/汇编 | 让操作系统在硬件(芯片/板子)上跑起来（驱动、内核、启动） |
| MCU固件工程师 | MCU底层开发与控制 | C/汇编 | 无OS或极简RTOS环境下实现硬件控制逻辑(相当于OS) |
| 嵌入式中间件/平台工程师 | 中间件与通用平台开发 | C/C++ | 封装通用功能模块（如通信协议栈、数据管理、日志系统、任务调度框架），解决跨硬件/跨系统的兼容性问题 |
| 算法/控制 | 算法设计 | C/C++ | 控制算法（PID、运动控制）、信号处理算法（滤波、FFT） |
| 嵌入式应用 | 业务功能 | C/C++ | 人机交互 |

* BSP工程师和MCU固件工程师的目标都是上层能使用硬件，只是运行的硬件规模和OS不同。

| 角色 | 硬件 | OS | 应用场景 |
| - | - | - | - |
| BSP工程师 | 工控机、复杂嵌入式主板等 | Linux/RTOS等 | 工业控制主机、医疗影像设备 |
| MCU固件工程师 | MCU | 无OS或极简RTOS | 传感器、家电控制板、小型执行器、便携式医疗设备 |

### 测试方法
1. 设备A视频传输到设备B显示的时延测试(2种)：
    1. 示波器：设备A和B的显示亮度都接入到示波器，A播放视频，看AB在示波器的时差
    1. 显示器：设备A的显示器上实时显示时间(到微秒)，设备B的显示器上也会显示时间。A和B放一起拍照，时间差值就是时延。因为显示器是一帧帧显示的(是误差)，注意显示器要高刷（如240Hz可降至≈4.2ms一帧）
